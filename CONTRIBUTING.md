# Contributing to SQL Visualizer

Thank you for your interest in contributing!

Please take a moment to review this document **before submitting a pull request**.

## Before You Start

It's never a fun experience to have your pull request declined after investing a lot of time and effort into a new feature. To avoid this from happening when creating a major features or architectural changes please:

1. [Open an issue first](https://github.com/stoftot/Itu-bsc-2026-sqlVisualizer/issues/new)
2. Explain the motivation
3. Discuss the proposed implementation

This helps avoid duplicated work and ensures new contributions align with the educational goals of the project.

 
## Project Philosophy

This project prioritizes:
- Educational clarity
- Strong mental models
- Visual understanding
- Interactive learning

The goal is NOT to perfectly simulate internal database engine behavior.

Some visualizations intentionally simplify or abstract SQL execution to improve learning outcomes.

Examples:
- GROUP BY is visualized as explicit grouping of rows
- Intermediate states are shown separately
- Animations prioritize readability over implementation detail

Please keep this philosophy in mind when contributing.


## Pull Request Guidelines

Before submitting a pull request:

- Ensure the project builds successfully
- Add tests when relevant
- Ensure that all test pass
- Keep pull requests focused and small
- Update documentation when needed

Pull requests should include:
- A clear description of the change
- Screenshots or videos for UI/visualization changes
- Explanation of educational impact if relevant


## Reporting Bugs

If you've noticed a bug, [please submit an issue](https://github.com/stoftot/Itu-bsc-2026-sqlVisualizer/issues/new).

When reporting bugs, please include:
- The SQL query used
- Expected behavior
- Actual behavior
- Error messages
- Screenshots if possible


## Feature Requests

Feature requests are welcome!

Please explain:
- The educational value
- The SQL concepts involved
- Example use cases
- Why the feature would improve understanding


## Good First Contributions

Beginner-friendly contribution ideas:
- Documentation fixes
- UI polish
- Animation improvements
- Bug reproduction tests
- Accessibility improvements
